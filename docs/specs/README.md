# Specs — per-task detail sheets

A spec is a task's stable contract — goal, scope edges, plan, and a
verifiable Done-means checklist — born just-in-time from the
discussion that opened the task ([TEMPLATE](TEMPLATE.md)); a
fully-specified [ROADMAP](../ROADMAP.md) line skips it. Naming:
`docs/specs/<id>-<short-name>.md`. Lifecycle, gates, and the weld:
[HOME §Task anatomy & lifecycle](../HOME.md#task-anatomy--lifecycle).
