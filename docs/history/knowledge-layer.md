# The knowledge layer: the vault learns to explain itself

## What shipped

A spec system for per-task detail sheets
([specs/](../specs/README.md)), a newcomer-proof
[GLOSSARY](../HOME.md#terms), a [DECISION-POLICY](../ENGINE.md)
hub consolidating every decided engine rule with sources, a
[SCHEMA](../data/SCHEMA.md) placeholder, [HOME](../HOME.md) rewritten as
the zero-state front page, YAML frontmatter across the corpus — and
merges no longer need a keypress: the founder's conversational yes
is the only merge gate →
[PR #58](https://github.com/wsher0901/roam/pull/58)

## Story

The knowledge layer in one PR: the per-task spec system, a
newcomer-proof GLOSSARY, DECISION-POLICY consolidating every decided
engine rule with source links (and surfacing five open engine questions
in its §10), a SCHEMA placeholder, HOME rewritten as the zero-state
front page, YAML frontmatter across the corpus — and the merge gate
went conversational: the founder's yes in chat became the only
keypress.
