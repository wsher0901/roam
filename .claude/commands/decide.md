description: Record a decision permanently in docs/DECISIONS.md
---
Take the decision we just discussed (or the one I describe now) and:
1. Append it to docs/DECISIONS.md as the next D-number with: date, the
   decision in one sentence, why (2-3 bullets), and alternatives rejected.
2. If the decision changes the product or plan, ALSO apply the edit in
   the same commit:
   - scope / functionality / principles -> docs/FOUNDATION.md
   - stages / tasks / timing -> docs/ROADMAP.md
   - good-but-not-now -> docs/IDEAS.md
3. Commit ("docs: record D-NNN <topic>") and push.
4. Finish by stating which of FOUNDATION / ROADMAP / DECISIONS changed
   and, if any did, remind me: "re-upload <file> to the Claude Web
   Project (or rely on live fetch)."
